# AGENTS.md

## Project Snapshot

- This is a Next.js 15 App Router project for `https://tools.mofei.life`.
- Package management is `pnpm`; keep `pnpm-lock.yaml` as the lockfile and do not reintroduce `package-lock.json`.
- Development runs through `pnpm dev`, which starts Next.js with Turbopack, HTTPS, and host `local.mofei.life`.
- Cloudflare deployment uses OpenNext via `@opennextjs/cloudflare`; keep Cloudflare-specific config in `open-next.config.ts`, `wrangler.jsonc`, and `next.config.ts`.
- The shared UI package `@mofei-dev/ui` is transpiled in `next.config.ts`.
- Current tool slugs are configured in [src/lib/site.ts](/Users/zhuwenlong/dev/mofei-dev-tools/src/lib/site.ts): `base64`, `bbox`, `geojson`, `json-extract`, `coordinate-converter`, and `objectid`.

## SEO And Routing Constraints

- English canonical routes use no language prefix.
  Examples: `/`, `/base64`, `/bbox`, `/geojson`, `/json-extract`, `/coordinate-converter`, `/objectid`
- Chinese canonical routes use `/zh/*`.
  Examples: `/zh`, `/zh/base64`, `/zh/bbox`, `/zh/geojson`, `/zh/json-extract`, `/zh/coordinate-converter`, `/zh/objectid`
- Do not switch English tool canonicals back to `/en/*`.
- `/en/*` routes still exist for localized access, but they are not the canonical English URLs for tool pages.
- Privacy pages follow the same canonical pattern: `/privacy` and `/zh/privacy`.
- BBox is a special multilingual tool. In addition to English and Chinese, it supports `/de/bbox`, `/es/bbox`, and `/fr/bbox`; maintain its hreflang set through [src/lib/bbox-i18n.ts](/Users/zhuwenlong/dev/mofei-dev-tools/src/lib/bbox-i18n.ts).
- Generic tool language routing is controlled by `SITE_LANGUAGES` in [src/lib/site.ts](/Users/zhuwenlong/dev/mofei-dev-tools/src/lib/site.ts). Do not add non-`en`/`zh` generic tool routes without updating that shared contract and the tests.

## Metadata Rules

- Keep canonical, hreflang, structured data URLs, sitemap, and robots consistent with the routing rules above.
- `metadataBase` is set in [src/app/layout.tsx](/Users/zhuwenlong/dev/mofei-dev-tools/src/app/layout.tsx).
- Tool page metadata is defined at the route layer, not duplicated in multiple places.
- For localized tool pages under `src/app/[lang]/*`, prefer the layout metadata files as the source of truth.
- Root tool pages should export concise route metadata with `createToolMetadata({ slug, ...getToolSeo(slug, 'en') })` or the equivalent shared SEO lookup.
- Localized ordinary tool layouts should validate `lang` with `isSiteLanguage()` before reading `TOOL_SEO`.
- BBox localized metadata should use `BBOX_SEO`, `bboxAlternates()`, `bboxUrl()`, and `bboxAlternateLocales()` from [src/lib/bbox-i18n.ts](/Users/zhuwenlong/dev/mofei-dev-tools/src/lib/bbox-i18n.ts).

## Sitemap And Robots

- Sitemap is generated by [src/app/sitemap.ts](/Users/zhuwenlong/dev/mofei-dev-tools/src/app/sitemap.ts).
- Robots is generated by [src/app/robots.ts](/Users/zhuwenlong/dev/mofei-dev-tools/src/app/robots.ts).
- Shared site constants live in [src/lib/site.ts](/Users/zhuwenlong/dev/mofei-dev-tools/src/lib/site.ts).
- Do not reintroduce static `public/sitemap.xml` or `public/robots.txt` unless there is a specific deployment reason.
- When adding a page that should be indexed, update the generated sitemap and robots allow list together.
- Keep BBox sitemap and robots entries generated from `BBOX_LANGUAGES`; do not hand-copy the language list in multiple files.

## URL Configuration

- Do not hardcode `https://tools.mofei.life/...` across pages or components.
- Use [src/lib/site.ts](/Users/zhuwenlong/dev/mofei-dev-tools/src/lib/site.ts) as the single source of truth for:
  `SITE_URL`, tool paths, absolute URLs, homepage URLs, tool URLs, and tool alternates.
- Prefer `toolUrl()`, `homeUrl()`, `absoluteUrl()`, and `toolAlternates()` over manual string concatenation.

## Page Structure

- Tool pages should follow the `PageComponent` pattern.
- Root routes and localized routes should be thin wrappers around the shared page component.
- Avoid making root routes depend on `src/app/[lang]/*/page.tsx` directly.
- Localized pages under `src/app/[lang]/*/page.tsx` should reject unsupported language params with `notFound()` and set `dynamicParams = false` when statically enumerated.
- Use `generateStaticParams()` from `SITE_LANGUAGES` for ordinary localized tools and from `BBOX_LANGUAGES` for BBox.
- `src/middleware.ts` passes the pathname through `x-pathname` so [src/app/layout.tsx](/Users/zhuwenlong/dev/mofei-dev-tools/src/app/layout.tsx) can set the correct `<html lang>`. Keep this behavior if route handling changes.
- Shared, reusable UI controls belong in `src/components/Common/` when they are used by more than one tool.

## Metadata Configuration

- Tool metadata skeletons should use [src/lib/metadata.ts](/Users/zhuwenlong/dev/mofei-dev-tools/src/lib/metadata.ts).
- Tool SEO copy lives in [src/lib/tool-seo.ts](/Users/zhuwenlong/dev/mofei-dev-tools/src/lib/tool-seo.ts).
- Keep route-level metadata concise by passing content into the shared metadata builder instead of repeating full metadata object structure.
- Root tool routes and localized tool layouts should both read from the shared SEO config instead of maintaining separate English and localized metadata strings.
- If you need to change common metadata shape such as `authors`, `alternates`, `openGraph`, or `twitter`, update the builder first.
- If you need to change tool-specific `title`, `description`, `keywords`, `category`, or `classification`, update `tool-seo.ts` first.
- BBox SEO copy lives in [src/lib/bbox-i18n.ts](/Users/zhuwenlong/dev/mofei-dev-tools/src/lib/bbox-i18n.ts) and is bridged into `TOOL_SEO` for the generic English/Chinese tool list.

## Homepage

- Homepage content should remain server-rendered and visible.
- Do not bring back the previous pattern where SEO text was hidden in `sr-only` content while the real homepage content was client-only.
- Homepage visible tool cards and homepage structured data should come from `getHomeTools()` in [src/lib/tool-content.ts](/Users/zhuwenlong/dev/mofei-dev-tools/src/lib/tool-content.ts).

## Tool Content Configuration

- Homepage tool cards and website-level tool JSON-LD should share the same tool copy source.
- Shared homepage/tool list content lives in [src/lib/tool-content.ts](/Users/zhuwenlong/dev/mofei-dev-tools/src/lib/tool-content.ts).
- When adding or renaming a tool, update shared tool content instead of editing homepage cards and structured data separately.
- Prefer config-driven updates over scattering per-component copies of tool names, descriptions, categories, and URLs.
- Keep `TOOL_SLUGS`, `TOOL_CONTENT`, `TOOL_SEO`, sitemap config, robots allow rules, and route folders aligned for every public tool.
- Navigation and footer language links should use shared path helpers instead of manual URL concatenation.

## Runtime And API Notes

- GeoJSON GitHub device flow proxies live in `src/app/api/github-device/route.ts` and `src/app/api/github-token/route.ts`.
- Keep API routes disallowed in robots unless there is a deliberate indexing reason.
- Google Analytics is configured through `NEXT_PUBLIC_GA_ID` and rendered from `src/components/GoogleAnalytics.tsx`.
- Do not expose secrets in client components or README examples.

## Verification

- Use `pnpm test` for the current Node test suite under `tests/`.
- Use `pnpm run build` before deployment or after metadata/routing changes with a broad impact.
- Use `pnpm run cf:build` when validating Cloudflare/OpenNext output.
- Existing tests assert SEO completeness, canonical route behavior, BBox multilingual routing, and sitemap/hreflang rules. Update tests when intentionally changing these contracts.

## Cleanup Guidance

- Before adding a new SEO helper or metadata file, check whether an existing route/layout already owns that responsibility.
- Remove dead SEO components or obsolete metadata files instead of leaving unused copies in the repo.
